## 1. 模块概述

### 1.1 定位

静态信息维护模块负责管理 MES 系统的**基础数据**，这些数据是生产执行的前置依赖：

- 用户与班组：生产执行所需的人力资源
- 产线与设备：生产所需的设备资源
- BOM 物料清单：产品结构定义
- 工序/工步/工艺流程：生产过程的工艺定义

### 1.2 模块关系

```
静态信息维护 (基础数据层)
  ├── 用户管理 ───── 班组管理 ───── 产线管理
  │                                    │
  ├── 设备管理 ───── 工步管理 ───── 工序管理 ───── 工艺流程管理
  │                                                    │
  └── BOM物料清单 ─────────────────────────────────────┘
                                                       │
三层生产架构 (生产执行层)                                 │
  Plan ──→ Order ──→ WorkOrder ────────────────────────┘
          (依赖产线)  (依赖工序+人员+产线)
```

### 1.3 通用设计模式

所有静态信息模块遵循统一的 CRUD 设计模式：

```
Controller
  ├── GET /{resource}          → 分页/全量查询
  ├── GET /{resource}/{id}     → 按ID查询
  ├── POST /{resource}         → 新增
  ├── PUT /{resource}          → 修改
  └── DELETE /{resource}       → 删除（批量/单体）
       │
       ▼
Service
  ├── CRUD 编排
  ├── 业务规则校验（唯一性、关联存在性）
  └── 级联删除处理
       │
       ▼
Mapper + XML
  └── 动态 SQL（if-test 非空字段更新）
```

## 2. 用户管理模块

### 2.1 模块说明

管理系统用户（员工）信息。用户同时是 RBAC 权限的主体，通过 `user_role` 关联角色，通过 `team_no` 关联班组。

### 2.2 实体: User

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| userName | String | 登录用户名（唯一） |
| password | String | 密码（当前明文存储，待加密） |
| name | String | 姓名 |
| position | String | 职位 |
| permissionLevel | String | 权限级别（已废弃，由 RBAC 替代） |
| createdAt | LocalDate | 创建日期 |
| updatedAt | LocalDateTime | 更新时间 |

### 2.3 接口清单

| 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|
| GET | /user | 条件分页查询用户 | 仅 LoginFilter |
| GET | /user/{id} | 按ID查询用户详情 | 仅 LoginFilter |
| GET | /user/join | 查询尚未加入班组的用户 | 仅 LoginFilter |
| POST | /user | 新增用户 | SYS_USER_CREATE |
| PUT | /user | 更新用户信息 | SYS_USER_UPDATE |
| DELETE | /user?ids=1,2,3 | 批量删除用户 | SYS_USER_DELETE |

### 2.4 接口详细设计

#### 条件分页查询用户

```
GET /user?page=1&pageSize=10&name=张&position=主管&begin=2026-01-01&end=2026-06-30
```

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 页码（默认1） |
| pageSize | Integer | 否 | 每页数量（默认10） |
| name | String | 否 | 姓名（模糊查询） |
| position | String | 否 | 岗位 |
| begin | String | 否 | 创建日期开始 |
| end | String | 否 | 创建日期结束 |

**响应**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "list": [
      {
        "id": 1,
        "userName": "admin",
        "name": "管理员",
        "position": "主管",
        "permissionLevel": "ADMIN"
      }
    ]
  }
}
```

#### 新增用户

```
POST /user
{
  "userName": "zhangsan",
  "password": "123456",
  "name": "张三",
  "position": "操作工",
  "permissionLevel": "USER"
}
```

**业务规则**:
- `userName` 必须唯一
- 密码建议使用 BCrypt 加密（当前为明文）
- 初始不分配角色，需通过 `/user-role` 接口单独分配

#### 批量删除用户

```
DELETE /user?ids=1,2,3
```

**级联行为**: 删除用户时，MySQL 外键级联删除 `user_role` 关联记录。业务层需补充缓存失效逻辑（如启用）。

## 3. 班组管理模块

### 3.1 模块说明

管理生产班组信息。班组关联产线（`line_no`），班长从用户表指定。

### 3.2 实体: WorkTeam

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| teamCode | String | 班组编号（唯一，如 `TEAM-A`） |
| teamName | String | 班组名称 |
| leaderId | Integer | 班组长用户ID |
| lineNo | String | 关联产线编号 |
| location | String | 班组位置 |

### 3.3 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /team | 查询所有班组 |
| GET | /team/{teamCode} | 按编号查询班组 |
| POST | /team | 新增班组 |
| PUT | /team/{teamCode} | 更新班组 |
| DELETE | /team/{teamCode} | 删除班组 |
| GET | /team/matrix | 查询技能矩阵 |
| POST | /team/matrix | 保存技能矩阵 |

### 3.4 接口详细设计

#### 查询所有班组

```
GET /team
```

**响应**:

```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "teamCode": "TEAM-A",
      "teamName": "装配A组",
      "leaderId": 1001,
      "leaderName": "王五",
      "memberCount": 15,
      "lineNo": "LINE-A01"
    }
  ]
}
```

#### 新增班组

```
POST /team
{
  "teamCode": "TEAM-C",
  "teamName": "焊接新组",
  "leaderId": 1010
}
```

**业务规则**: `teamCode` 必须全局唯一。

#### 技能矩阵

技能矩阵记录每个员工在每道工序上的技能掌握情况（是否擅长），即 `skills` 表：

```
GET /team/matrix?teamCode=TEAM-A
```

**响应**:

```json
{
  "code": 200,
  "data": [
    {
      "userName": "zhangsan",
      "name": "张三",
      "processName": "焊接",
      "choose": 1,
      "teamNo": "TEAM-A"
    }
  ]
}
```

## 4. 产线管理模块

### 4.1 模块说明

管理生产产线（流水线/工作站）。产线是 Plan → Order 拆分的关键维度，同时也是班组绑定的物理位置。产线可绑定工艺流程（`process_flow_id`）。

### 4.2 实体: ProductionLine

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| lineNo | String | 产线编号（唯一，如 `LINE-A01`） |
| lineName | String | 产线名称 |
| status | String | 状态（空闲/运行/暂停） |
| processFlowId | Integer | 当前执行的工艺流程ID |

### 4.3 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /line | 查询所有产线 |
| GET | /line/{lineNo} | 按编号查询产线 |
| POST | /line | 新增产线 |
| PUT | /line/{lineCode} | 更新产线 |
| DELETE | /line/{lineNo} | 删除产线 |
| GET | /line/compose/{lineNo} | 查询产线班组配置 |
| PUT | /line/compose | 保存产线班组配置 |

### 4.4 接口详细设计

#### 查询所有产线

```
GET /line
```

**响应**: 返回所有产线的基础信息列表。

#### 新增产线

```
POST /line
{
  "lineNo": "LINE-C01",
  "lineName": "装配三线",
  "status": "空闲"
}
```

#### 产线班组配置

维护产线与班组的绑定关系（通过 `work_teams.line_no` 实现）：

```
GET /line/compose/LINE-A01
```

**响应**: 返回该产线下的班组列表及相关配置。

## 6. BOM物料清单模块

### 6.1 模块说明

管理产品物料清单（Bill of Materials）。通过 `parent_id` 字段支持**树形层级结构**（自引用）。

### 6.2 实体: Bom

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| drawingNo | String | 图号 |
| nameSpecification | String | 名称规格 |
| material | String | 材质 |
| quantity | Integer | 数量 |
| unitWeight | BigDecimal | 单重 |
| type | String | 类型 |
| parentId | Integer | 父物料ID（NULL=根节点） |

### 6.3 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /bom | 查询所有BOM（树形结构） |
| GET | /bom/{id} | 按ID查询BOM |
| POST | /bom | 新增BOM |
| PUT | /bom | 更新BOM |
| PUT | /bom/level | 更新BOM层级（父子关系） |
| DELETE | /bom/{ids} | 批量删除BOM（路径参数，逗号分隔） |

### 6.4 接口详细设计

#### 查询所有BOM（树形）

```
GET /bom
```

**响应**: 返回完整的物料树结构（递归查询或一次查询+前端组装）。

#### 新增BOM

```
POST /bom
{
  "drawingNo": "DWG-001",
  "nameSpecification": "车载摄像头模组总成",
  "material": "铝+电子元件",
  "quantity": 1,
  "unitWeight": 0.250,
  "type": "成品",
  "parentId": null
}
```

#### 更新BOM层级

```
PUT /bom/level
{
  "id": 5,
  "parentId": 1
}
```

**业务规则**: 将物料 #5 的父物料修改为 #1，不能形成循环引用。

#### 删除BOM

```
DELETE /bom/1,2,3
```

**级联行为**: MySQL CASCADE 会自动删除关联的工艺流程、input/output 记录。

## 7. 工序管理模块

### 7.1 模块说明

管理生产工序定义。工序是工艺流程的基本构成单元，每道工序定义了计划工时、质检指标等。

### 7.2 实体: Process

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| processName | String | 工序名称（唯一） |
| processCode | String | 工序编码 |
| plannedWorkingHours | Double | 计划工时（小时） |
| description | String | 工序描述 |
| qualityControlPoint | String | 质检指标 |

### 7.3 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /process?page=&pageSize=&processName=&processCode= | 条件分页查询 |
| GET | /process/listAll | 查询所有工序（不分页） |
| GET | /process/{processId} | 按ID查询工序名称 |
| POST | /process | 新增工序 |
| PUT | /process | 更新工序 |
| DELETE | /process?ids=1,2 | 批量删除工序 |

### 7.4 接口详细设计

#### 条件分页查询

```
GET /process?page=1&pageSize=10&processName=焊接&processCode=WELD
```

#### 新增工序

```
POST /process
{
  "processName": "SMT贴片",
  "plannedWorkingHours": 2.5,
  "description": "表面贴装焊接工序",
  "qualityControlPoint": "焊点饱满，无虚焊，偏移量<0.1mm"
}
```

**业务规则**: `processName` 必须全局唯一。

#### 查询工序用途

工序被三层生产架构中的 WorkOrder 引用。此外：
- `process_sequences` 定义了工序在工艺流程中的前后序
- `process_steps` 关联了工序的具体作业步骤
- `skills` 关联了员工对该工序的技能掌握情况
- `input`/`output` 定义了工序的物料投入产出

## 9. 工艺流程管理模块

### 9.1 模块说明

管理工艺流程。工艺流程将多道工序按**前后序关系**组织起来，定义了一个完整的产品加工路径。工艺流程关联 BOM（`bom_id`），通过 `process_flow_processes` 表关联多道工序，通过 `process_sequences` 定义执行顺序。

### 9.2 实体: ProcessFlow

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| bomId | Integer | 关联BOM |
| flowName | String | 流程名称 |
| status | String | 状态（有效/废弃） |
| plannedWorkingHours | Integer | 计划总工时 |

### 9.3 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /flow | 查询所有工艺流程 |
| GET | /flow/chart/{id} | 查询工艺流程图形化数据 |
| POST | /flow/chart | 保存工艺流程图配置 |
| POST | /flow | 新增工艺流程 |
| PUT | /flow | 更新工艺流程 |
| DELETE | /flow?id= | 删除工艺流程 |

### 9.4 接口详细设计

#### 查询所有工艺流程

```
GET /flow
```

#### 新增工艺流程

```
POST /flow
{
  "bomId": 1001,
```

## 附录 A: 模块间数据依赖关系

```
bom (物料清单)
  ├── process_flows (工艺流程)
  │     ├── process_flow_processes (关联工序)
  │     │     └── processes (工序定义)
  │     │           ├── process_sequences (工序前后序)
  │     │           ├── process_steps (工序-步骤关联)
  │     │           │     └── work_steps (工步)
  │     │           │           ├── equipment_info (设备)
  │     │           │           └── equipment_functions (设备功能)
  │     │           ├── input / output (投产物料)
  │     │           └── skills (员工技能)
  │     └── production_lines (产线绑定)
  │
production_plan (生产计划)
  ├── 引用 bom.id
  └── 发布时通过 process_flows 查找工艺流程 → 产线 → 生成订单

production_order (生产订单)
  ├── 引用 plan.planNo + production_lines.lineNo
  └── 发布时通过产线工艺流程 → 工序 → 人员 → 生成工单

work_order (工单)
  ├── 引用 order.orderNo + processes.id + users.id
  └── 通过 is_critical 标记影响订单状态聚合
```
