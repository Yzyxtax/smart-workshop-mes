### 2.3 RBAC 权限域

```
┌──────────────┐     ┌───────────────┐     ┌─────────────┐
│     role     │     │   permission  │     │    users    │
│  id (PK)     │     │  id (PK)      │     │  id (PK)    │
│  role_code   │     │  permission_  │     │  username   │
│  role_name   │     │    code (UK)  │     │  name       │
│  description │     │  permission_  │     └──────┬──────┘
└──────┬───────┘     │    name       │            │
       │             │  module       │            │
       │             │  description  │            │
       │             └───────┬───────┘            │
       │                     │                    │
       ▼                     ▼                    ▼
┌──────────────────┐  ┌──────────────────┐  ┌──────────────┐
│ role_permission  │  │    user_role     │  │ (关联方向:    │
│  role_id (FK)    │  │  user_id (FK)    │  │  user → role │
│  permission_id   │  │  role_id (FK)    │  │  role → perm │
│  (FK)            │  │  created_at      │  │  两步 JOIN)  │
│  uk_role_perm    │  │  uk_user_role    │  └──────────────┘
└──────────────────┘  └──────────────────┘

权限查询路径（SQL）:
  SELECT DISTINCT p.permission_code
  FROM users u
  JOIN user_role ur ON u.id = ur.user_id
  JOIN role_permission rp ON ur.role_id = rp.role_id
  JOIN permission p ON rp.permission_id = p.id
  WHERE u.id = #{userId}
```

## 3. 核心业务表设计

### 3.1 bom — 物料清单

> 支持树形层级结构，通过 `parent_id` 自引用实现。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK, AUTO_INCREMENT | 物料主键 |
| drawing_no | varchar(100) | 可空 | 图号 |
| name_specification | varchar(200) | 可空 | 名称规格 |
| material | varchar(100) | 可空 | 材质 |
| quantity | int | 可空 | 数量 |
| unit_weight | decimal(10,3) | 可空 | 单重 |
| type | varchar(10) | 可空 | 类型 |
| parent_id | int | FK(bom.id), 可空 | 父物料ID |

**索引**: 主键 `id`  
**外键**: `parent_id` → `bom(id)` ON UPDATE CASCADE ON DELETE CASCADE

**业务规则**:
- `parent_id` 为 NULL 表示根物料
- 通过 `GET /bom` 获取完整物料树（递归查询）

### 3.2 equipment_info — 设备信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK, AUTO_INCREMENT | 设备主键 |
| name | varchar(100) | NOT NULL | 设备名称 |
| type | varchar(50) | 可空 | 设备类型 |
| model | varchar(100) | 可空 | 型号 |
| production_date | date | 可空 | 生产日期 |
| manufacturer | varchar(100) | 可空 | 制造商 |

**索引**: `idx_ei_name` on `name`

### 3.3 equipment_functions — 设备功能

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK, AUTO_INCREMENT | 功能主键 |
| equipment_info_id | int | FK NOT NULL | 关联设备ID |
| function_description | text | 可空 | 功能描述 |

**外键**: `equipment_info_id` → `equipment_info(id)` CASCADE

### 3.4 process_flows — 工艺流程

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK, AUTO_INCREMENT | 流程主键 |
| bom_id | int | FK NOT NULL | 关联 BOM |
| flow_name | varchar(100) | NOT NULL | 流程名称 |
| status | varchar(10) | NOT NULL | 状态（有效/废弃） |
| planned_working_hours | int | 可空 | 计划总工时 |

**外键**: `bom_id` → `bom(id)` CASCADE

**业务规则**: 发布门禁中校验 "工艺流程存在性"，通过此表检查 BOM 是否有关联的有效工艺流程。

### 3.5 processes — 工序定义

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK, AUTO_INCREMENT | 工序主键 |
| process_name | varchar(100) | NOT NULL, UNIQUE | 工序名称 |
| planned_working_hours | double | 可空 | 计划工时（小时） |
| description | text | 可空 | 工序描述 |
| quality_control_point | text | 可空 | 质检指标 |

**唯一约束**: `process_name`  
**索引**: `idx_p_name` on `process_name`

### 3.6 input / output — 工序投入/产出物料

| 表名 | 字段 | 类型 | 约束 |
|------|------|------|------|
| input | id | int | PK |
| input | input | int | FK → bom(id) |
| input | process_id | int | FK → processes(id) |
| output | id | int | PK |
| output | output | int | FK → bom(id) |
| output | process_id | int | FK → processes(id) |

**含义**: 每道工序需要投入哪些物料（input），产出哪些物料（output）。

### 3.7 process_flow_processes — 工艺流程-工序关联

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| process_flow_id | int | FK NOT NULL | 工艺流程ID |
| process_id | int | FK NOT NULL | 工序ID |

**索引**: `process_flow_id`, `process_id`

### 3.8 process_sequences — 工序顺序

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| process_id | int | FK NOT NULL | 当前工序 |
| previous_process_id | int | FK, 可空 | 前驱工序 |
| process_flow_id | int | FK NOT NULL | 所属工艺流程 |

**含义**: 定义工艺流程内各工序的前驱→后继执行顺序。`previous_process_id` 为 NULL 表示首工序。

### 3.9 production_lines — 产线

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| line_no | varchar(50) | NOT NULL, UNIQUE | 线别编号 |
| line_name | varchar(100) | NOT NULL | 线别名称 |
| line_status | varchar(20) | DEFAULT '空闲' | 状态（空闲/运行/暂停） |
| process_flow_id | int | FK, 可空 | 当前执行的工艺流程 |

**外键**: `process_flow_id` → `process_flows(id)` CASCADE

**业务规则**: 发布门禁中校验 "产线可用性"，查询 `line_status = '空闲'` 的产线。

### 3.10 work_steps — 作业步骤

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| name | varchar(100) | NOT NULL | 步骤名称 |
| equipment_id | int | FK, 可空 | 关联设备 |
| function_id | int | FK, 可空 | 关联设备功能 |
| description | text | 可空 | 步骤描述 |

### 3.11 process_steps — 工序-步骤关联

| 字段名 | 类型 | 约束 |
|--------|------|------|
| id | int | PK |
| process_id | int | FK → processes(id) |
| work_step_id | int | FK → work_steps(id) |

### 3.12 work_teams — 班组

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| team_name | varchar(100) | NOT NULL | 班组名称 |
| team_location | varchar(200) | 可空 | 班组位置 |
| line_no | varchar(50) | FK, 可空 | 关联产线 |
| team_no | varchar(50) | NOT NULL, UNIQUE | 班组编号 |

**外键**: `line_no` → `production_lines(line_no)` CASCADE

### 3.13 users — 用户（员工）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK, AUTO_INCREMENT | 用户主键 |
| username | varchar(50) | NOT NULL, UNIQUE | 登录用户名 |
| password | varchar(255) | NOT NULL, DEFAULT '123456' | 密码（明文存储） |
| name | varchar(50) | NOT NULL | 真实姓名 |
| position | varchar(50) | 可空 | 职位 |
| created_at | date | DEFAULT 当前日期 | 创建日期 |
| updated_at | timestamp | DEFAULT CURRENT_TIMESTAMP | 更新时间戳 |
| team_no | varchar(50) | FK, 可空 | 所属班组编号 |

> ⚠️ **安全提示**: 当前密码为明文存储（`varchar(255)`），生产环境应迁移至 BCrypt 加密。

### 3.14 skills — 员工技能

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| username | varchar(50) | FK NOT NULL | 用户名 |
| name | varchar(50) | 可空 | 员工姓名（冗余） |
| process_name | varchar(100) | FK NOT NULL | 工序名称 |
| choose | tinyint(1) | NOT NULL, DEFAULT 0 | 是否擅长 |
| team_no | varchar(50) | FK, 可空 | 班组编号 |

**业务规则**: 发布门禁中校验 "人员可执行性"，通过此表查询具备某工序技能的员工。

## 4. RBAC 权限表设计

### 4.1 permission — 权限表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | 权限ID |
| permission_code | varchar(100) | NOT NULL, UNIQUE | 权限编码 |
| permission_name | varchar(100) | NOT NULL | 权限名称（中文） |
| module | varchar(50) | NOT NULL | 所属模块 |
| description | text | 可空 | 权限描述 |
| created_at | datetime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**权限编码格式**: `{MODULE}_{ACTION}`，如 `ORDER_CREATE`、`SYS_USER_DELETE`

**模块枚举**: SYS、PLAN、ORDER、WORKORDER、EQU、BOM、PROC

### 4.2 role — 角色表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | 角色ID |
| role_code | varchar(50) | NOT NULL, UNIQUE | 角色编码 |
| role_name | varchar(100) | NOT NULL | 角色名称 |
| description | text | 可空 | 角色描述 |
| created_at | datetime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | datetime | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**角色编码格式**: `ROLE_{DOMAIN}_{ROLE_NAME}`，如 `ROLE_PROD_MANAGER`

### 4.3 role_permission — 角色权限关联

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| role_id | int | FK NOT NULL | 角色ID |
| permission_id | int | FK NOT NULL | 权限ID |
| created_at | datetime | DEFAULT CURRENT_TIMESTAMP | — |

**唯一约束**: `uk_role_permission(role_id, permission_id)`

### 4.4 user_role — 用户角色关联

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | int | PK | — |
| user_id | int | FK NOT NULL | 用户ID |
| role_id | int | FK NOT NULL | 角色ID |
| created_at | datetime | DEFAULT CURRENT_TIMESTAMP | — |

**唯一约束**: `uk_user_role(user_id, role_id)`

| old_status | enum(同 Plan) | NOT NULL | 原状态 |
| new_status | enum(同 Plan) | NOT NULL | 新状态 |
| operator_id | int | FK NOT NULL | 操作人 ID |

**外键**: `operator_id` → `users(id)` CASCADE

## 7. 外键与级联规则

### 7.1 级联策略

所有外键统一采用 `ON UPDATE CASCADE ON DELETE CASCADE`：

| 场景 | 级联行为 | 示例 |
|------|----------|------|
| 删除 BOM | 级联删除关联的工艺流程、input/output | DELETE bom → CASCADE process_flows |
| 删除产线 | 级联删除关联的班组绑定 | DELETE production_lines → CASCADE work_teams.line_no |
| 删除工艺 | 级联删除关联的工序关联、工序顺序 | DELETE process_flows → CASCADE process_flow_processes |
| 删除用户 | 级联删除 user_role 关联 | DELETE users → CASCADE user_role |
| 删除角色 | 级联删除 role_permission 关联 | DELETE role → CASCADE role_permission |
| 删除计划 | 级联删除关联订单 | DELETE production_plan → CASCADE production_order |
| 删除订单 | 级联删除关联工单 | DELETE production_order → CASCADE work_order |

### 7.2 级联操作中业务层补充

MySQL 级联删除属于物理层操作，业务层需补充：
- **删除用户时**: 级联失效 JWT Token（如启用缓存）
- **删除角色时**: 需手动将拥有该角色的用户权限缓存失效
- **删除计划时**: 需审计日志记录（级联删除的订单/工单审计由业务层记录）

## 9. 编号生成规则

### 9.1 计划编号 (plan_no)

**格式**: `PLAN-yyyyMMdd-nnn`

- `PLAN`: 固定前缀
- `yyyyMMdd`: 创建日期（如 `20260608`）
- `nnn`: 当日自增序号（如 `001`）
- 示例: `PLAN-20260608-001`

### 9.2 订单编号 (order_no)

**格式**: `{planNo}-{lineNo}`

- `{planNo}`: 所属计划编号
- `{lineNo}`: 产线编号
- 示例: `PLAN-20260608-001-LINE-A01`

### 9.3 工单编号 (work_order_no)

**格式**: `{orderNo}-{processId}-{seq}`

- `{orderNo}`: 所属订单编号
- `{processId}`: 工序 ID
- `{seq}`: 同一订单+工序下的自增序号（从 1 开始）
- 示例: `PLAN-20260608-001-LINE-A01-10-1`

## 10. 数据一致性保证

### 10.1 事务边界

所有写操作（CRUD 的 CUD + 状态变更）均在 Spring Boot 声明式事务中执行：
- `@Transactional` 注解标注在服务层方法上
- 默认隔离级别：READ_COMMITTED
- 回滚策略：RuntimeException 及其子类

### 10.2 状态一致性

1. **级联操作中的状态变更**：使用直接 DB 更新避免递归回写
2. **状态聚合**：Order → Plan 的状态同步通过 O(N) 单次遍历实现
3. **关键工单标记**：仅 `is_critical = true` 的工单参与订单状态聚合
4. **终态保护**：状态机 + 业务层双重校验，防止状态回退

### 10.3 编号唯一性

1. **数据库约束**: plan_no、order_no 使用 UNIQUE 约束
2. **业务层校验**: 同一计划+产线、同一订单+工序+人员不可重复创建

### 10.4 审计完整性

- 所有状态变更（包括级联触发的变更）均写入 `status_history` 表
- 审计记录包含：对象类型、对象编号、原状态、新状态、操作人、操作时间
- 审计记录不可修改/删除
