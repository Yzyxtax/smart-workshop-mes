## 1. 文档概述

### 1.1 文档目的

本文档为智联车间轻量化MES系统的需求规格说明书，旨在完整、准确地描述系统的功能需求、非功能需求、接口需求及数据需求，作为系统分析、设计、开发、测试与验收的依据。

本文档面向以下读者：

- 项目管理人员：用于项目计划与进度控制
- 系统架构师：用于技术方案设计与系统架构决策
- 开发人员：用于详细设计与编码实现
- 测试人员：用于测试用例设计与测试执行
- 业务人员：用于需求确认与验收标准判定

### 1.2 项目背景

智联车间轻量化MES系统面向中小型制造企业，围绕生产过程的数字化管理，提升生产计划与执行效率、人员协同与工单流转效率、生产质量预测与问题定位能力、生产过程透明化与实时可视化能力，以及智能化排产与决策能力。

系统强调**轻量部署、敏捷迭代、开源技术栈、低成本运维**的核心理念，采用前后端分离+模块化单体架构设计，支持按需扩展与未来向微服务架构演进。

## 2. 系统概述

### 2.1 系统目标

智联车间轻量化MES系统旨在为中小型制造企业提供一套轻量化、可扩展的生产执行管理系统，实现以下核心目标：

1. **生产计划与执行效率提升**：通过数字化管理生产计划、订单和工单，减少人工干预，提高计划执行率
2. **人员协同与工单流转效率提升**：优化工单分派、报工流程，实现生产过程的流畅运转
3. **生产质量预测与问题定位能力**：基于历史数据实现质量预测和根因分析
4. **生产过程透明化与实时可视化**：通过实时看板展示生产进度，支持秒级刷新
5. **智能化排产与决策能力**：集成AI算法实现自动排产，辅助生产决策

### 2.2 系统边界

#### 2.2.1 系统范围

本系统的功能范围包括：

- 基础数据管理（BOM、设备、用户、组织结构）
- 工艺流程管理（工步、工序、工艺流程设计）
- 生产计划管理（计划制定、发布、执行、监控）
- 生产订单管理（订单拆分、分配、状态跟踪）
- 工单管理（工单分派、执行、报工、完工）
- 实时看板与预警
- 权限管理与日志审计

#### 2.2.2 系统边界外

以下功能不在本系统范围内：

- ERP系统的财务核算功能
- SCM系统的供应链管理功能
- WMS系统的仓库管理功能
- 设备PLC直接控制功能

用户权限模块是系统的基础模块，负责管理系统用户、组织结构、角色权限，为其他功能模块提供权限控制基础。

| 角色编码                      | 角色名称  | 职责描述                       | 权限范围         |
| ------------------------- | ----- | -------------------------- | ------------ |
| ROLE\_PROD\_MANAGER       | 生产主管  | 管理生产计划、生产订单、查看看板、执行计划发布与暂停 | 生产计划、订单、看板查看 |
| ROLE\_PROCESS\_ENGINEER   | 工艺工程师 | 管理BOM、工步、工序、工艺流程           | 工艺设计全部功能     |
| ROLE\_HR\_MANAGER         | 人事主管  | 管理员工、班组、产线、技能矩阵            | 人员组织管理全部功能   |
| ROLE\_WORKSHOPS\_DIRECTOR | 车间主任  | 管理生产订单、工单、查看进度与预警          | 订单、工单、进度查看   |
| ROLE\_TEAM\_LEADER        | 班组长   | 管理工单分派、报工策略、工单执行           | 本班组工单管理      |
| ROLE\_WORKER              | 工人    | 开工、报工、完工                   | 本工单执行与报工     |

#### 3.2.3 功能需求

**用户管理（F-SYS-001）**

| 功能点  | 需求描述                     | 优先级 |
| ---- | ------------------------ | --- |
| 用户注册 | 支持新用户注册，包含用户名、密码、姓名、岗位信息 | P0  |
| 用户登录 | 支持用户名密码登录，返回JWT令牌        | P0  |
| 用户查询 | 支持分页查询用户列表，支持按姓名、岗位筛选    | P1  |
| 用户更新 | 支持更新用户基本信息               | P1  |
| 用户删除 | 支持删除用户（逻辑删除）             | P2  |
| 密码修改 | 支持修改当前用户密码               | P1  |

**接口规格**

| 接口路径           | 方法     | 说明     | 权限要求        |
| -------------- | ------ | ------ | ----------- |
| /api/user      | GET    | 查询用户列表 | ROLE\_ADMIN |
| /api/user/{id} | GET    | 查询用户详情 | 登录用户        |
| /api/user      | POST   | 创建用户   | ROLE\_ADMIN |
| /api/user      | PUT    | 更新用户   | ROLE\_ADMIN |
| /api/user/{id} | DELETE | 删除用户   | ROLE\_ADMIN |
| /api/login     | POST   | 用户登录   | 公开          |

**数据模型**

```
User {
    id: Integer           // 用户ID，主键
    userName: String      // 用户名，唯一
    password: String      // 密码（加密存储）
    name: String          // 姓名
    position: String      // 岗位
    permissionLevel: String // 权限级别
    createdAt: Date       // 创建时间
    updatedAt: DateTime   // 更新时间
}
```

#### 3.6.3 状态机定义

**状态流转**

```
CREATED（创建）→ RELEASED（发布）→ RUNNING（执行）→ PAUSED（暂停）→ RUNNING（执行）→ COMPLETED（完成）
                          ↓
                      PAUSED（暂停）
```

**可用动作**

| 动作              | 说明   | 触发条件       |
| --------------- | ---- | ---------- |
| PUBLISH         | 发布   | CREATED状态  |
| CANCEL\_PUBLISH | 取消发布 | RELEASED状态 |
| START\_WORK     | 开始执行 | RELEASED状态 |
| PAUSE           | 暂停   | RUNNING状态  |
| RESUME          | 恢复执行 | PAUSED状态   |
| FINISH\_WORK    | 完成   | RUNNING状态  |
